# Glossaire Cloud Fundamentals

Ce glossaire regroupe les termes essentiels du cloud computing. Utilisez-le comme référence rapide pendant et après la formation.

---

## A

| Terme | Définition |
|-------|------------|
| **API** | Application Programming Interface - Interface permettant à deux applications de communiquer |
| **AutoML** | Machine Learning automatisé où le service sélectionne automatiquement les meilleurs algorithmes |
| **Auto Scaling** | Ajustement automatique du nombre de ressources selon la charge |
| **Availability Zone (AZ)** | Datacenter isolé au sein d'une région cloud |

## B

| Terme | Définition |
|-------|------------|
| **Bare Metal** | Serveur physique dédié, sans virtualisation |
| **Blob Storage** | Stockage d'objets binaires (Binary Large Object) - Nom Azure pour Object Storage |
| **Block Storage** | Stockage en blocs, équivalent d'un disque dur (EBS, Managed Disks) |
| **BYOL** | Bring Your Own License - Utiliser ses propres licences sur le cloud |

## C

| Terme | Définition |
|-------|------------|
| **CaaS** | Container as a Service - Service de conteneurs managé |
| **CapEx** | Capital Expenditure - Dépenses d'investissement (achat serveurs) |
| **CDN** | Content Delivery Network - Réseau de distribution de contenu |
| **CI/CD** | Continuous Integration / Continuous Delivery - Automatisation du build et déploiement |
| **Cloud Native** | Application conçue pour le cloud (microservices, conteneurs, élastique) |
| **Cluster** | Ensemble de machines travaillant ensemble (ex: Kubernetes cluster) |
| **Compliance** | Conformité aux normes et réglementations |
| **Container** | Environnement isolé contenant une application et ses dépendances |
| **Cutover** | Bascule finale lors d'une migration |

## D

| Terme | Définition |
|-------|------------|
| **Data Lake** | Stockage de données brutes, tous formats |
| **Data Warehouse** | Base de données optimisée pour l'analyse et le reporting |
| **DevOps** | Culture et pratiques rapprochant développement et opérations |
| **DevSecOps** | DevOps avec intégration de la sécurité à toutes les étapes |
| **DNS** | Domain Name System - Résolution des noms de domaine |
| **Docker** | Plateforme de conteneurisation |
| **DORA** | DevOps Research and Assessment - Métriques de performance DevOps |
| **DR** | Disaster Recovery - Reprise après sinistre |

## E

| Terme | Définition |
|-------|------------|
| **Edge Computing** | Traitement des données au plus près de la source |
| **Egress** | Trafic sortant (souvent facturé dans le cloud) |
| **Elasticity** | Capacité à adapter automatiquement les ressources à la demande |
| **ETL** | Extract, Transform, Load - Pipeline de données |
| **ELT** | Extract, Load, Transform - Variante moderne où la transformation se fait après chargement |

## F

| Terme | Définition |
|-------|------------|
| **FaaS** | Function as a Service - Exécution de code sans serveur (Lambda, Functions) |
| **Feature Store** | Base de features réutilisables pour le ML |
| **FinOps** | Financial Operations - Optimisation des coûts cloud |
| **Firewall** | Pare-feu filtrant le trafic réseau |
| **Foundation Model** | Grand modèle IA pré-entraîné (GPT, PaLM, Claude) |

## G

| Terme | Définition |
|-------|------------|
| **Gateway** | Point d'entrée/sortie d'un réseau |
| **Generative AI** | IA capable de créer du contenu (texte, images, code) |
| **Git** | Système de contrôle de version distribué |
| **GitOps** | Gestion de l'infrastructure via Git |

## H

| Terme | Définition |
|-------|------------|
| **HA** | High Availability - Haute disponibilité |
| **HSM** | Hardware Security Module - Module matériel de sécurité cryptographique |
| **Hybrid Cloud** | Combinaison de cloud public et privé |
| **Hypervisor** | Logiciel permettant la virtualisation (VMware, Hyper-V, KVM) |

## I

| Terme | Définition |
|-------|------------|
| **IaaS** | Infrastructure as a Service - VMs, stockage, réseau |
| **IAM** | Identity and Access Management - Gestion des identités et accès |
| **IaC** | Infrastructure as Code - Définition de l'infra dans du code (Terraform) |
| **Image** | Template pour créer des conteneurs ou VMs |
| **Ingress** | Trafic entrant / Contrôleur d'entrée Kubernetes |
| **Instance** | VM ou ressource compute individuelle |

## K

| Terme | Définition |
|-------|------------|
| **Kubernetes (K8s)** | Orchestrateur de conteneurs open source |
| **KPI** | Key Performance Indicator - Indicateur clé de performance |

## L

| Terme | Définition |
|-------|------------|
| **Latency** | Temps de réponse / délai |
| **Lift & Shift** | Migration telle quelle sans modification (Rehost) |
| **LLM** | Large Language Model - Grand modèle de langage |
| **Load Balancer** | Répartiteur de charge |

## M

| Terme | Définition |
|-------|------------|
| **Managed Service** | Service géré par le provider (pas d'admin serveur) |
| **Microservices** | Architecture où l'application est découpée en petits services indépendants |
| **MFA** | Multi-Factor Authentication - Authentification multi-facteurs |
| **ML** | Machine Learning - Apprentissage automatique |
| **MLOps** | DevOps appliqué au Machine Learning |
| **MTTR** | Mean Time To Recovery - Temps moyen de récupération |
| **Multi-Cloud** | Utilisation de plusieurs providers cloud |
| **Multi-Tenant** | Architecture où plusieurs clients partagent la même infrastructure |

## N

| Terme | Définition |
|-------|------------|
| **NACL** | Network Access Control List - ACL réseau (AWS) |
| **Namespace** | Isolation logique dans Kubernetes |
| **NAT** | Network Address Translation - Translation d'adresses réseau |
| **NIST** | National Institute of Standards and Technology - Définit les standards cloud |
| **Node** | Machine dans un cluster Kubernetes |

## O

| Terme | Définition |
|-------|------------|
| **Object Storage** | Stockage d'objets (S3, Blob Storage, Cloud Storage) |
| **On-Demand** | Facturation à l'usage, sans engagement |
| **On-Premise** | Infrastructure hébergée dans ses propres locaux |
| **OpEx** | Operational Expenditure - Dépenses d'exploitation |
| **Orchestration** | Automatisation et coordination de services (Kubernetes) |

## P

| Terme | Définition |
|-------|------------|
| **PaaS** | Platform as a Service - Plateforme de développement managée |
| **PCI-DSS** | Payment Card Industry Data Security Standard |
| **Pipeline** | Séquence automatisée d'étapes (CI/CD, data) |
| **Pod** | Plus petite unité déployable dans Kubernetes |
| **Private Cloud** | Cloud dédié à une seule organisation |
| **Public Cloud** | Cloud partagé, accessible à tous (AWS, Azure, GCP) |

## R

| Terme | Définition |
|-------|------------|
| **RBAC** | Role-Based Access Control - Contrôle d'accès basé sur les rôles |
| **Refactor** | Réécrire une application pour le cloud natif |
| **Region** | Zone géographique contenant plusieurs AZ |
| **Registry** | Dépôt d'images (Docker Hub, ECR, ACR) |
| **Rehost** | Migration lift & shift sans modification |
| **Replatform** | Migration avec optimisations mineures |
| **Reserved Instance** | Instance avec engagement (1-3 ans) pour réduction |
| **Rollback** | Retour à une version précédente |
| **RPO** | Recovery Point Objective - Perte de données acceptable |
| **RTO** | Recovery Time Objective - Temps d'arrêt acceptable |

## S

| Terme | Définition |
|-------|------------|
| **SaaS** | Software as a Service - Application complète en ligne |
| **SAST** | Static Application Security Testing - Analyse de sécurité du code |
| **Scaling** | Ajustement de capacité (vertical: taille, horizontal: nombre) |
| **Security Group** | Firewall virtuel au niveau instance |
| **Self-Healing** | Capacité à se réparer automatiquement |
| **Serverless** | Exécution sans gestion de serveurs (Lambda, Functions) |
| **Service Mesh** | Infrastructure pour la communication entre microservices |
| **SLA** | Service Level Agreement - Niveau de service garanti |
| **Spot Instance** | Instance à prix réduit, interruptible |
| **Stateless** | Application sans état local (requis pour scaling horizontal) |
| **Subnet** | Sous-réseau dans un VPC |

## T

| Terme | Définition |
|-------|------------|
| **Tagging** | Étiquetage des ressources pour organisation et facturation |
| **TCO** | Total Cost of Ownership - Coût total de possession |
| **Terraform** | Outil IaC multi-cloud |
| **TLS** | Transport Layer Security - Chiffrement des communications |
| **Tokenization** | Remplacement de données sensibles par des tokens |

## V

| Terme | Définition |
|-------|------------|
| **VM** | Virtual Machine - Machine virtuelle |
| **VNet** | Virtual Network - Réseau virtuel (Azure) |
| **VPC** | Virtual Private Cloud - Réseau privé virtuel |
| **VPN** | Virtual Private Network - Réseau privé virtuel |

## W

| Terme | Définition |
|-------|------------|
| **WAF** | Web Application Firewall - Pare-feu applicatif web |
| **Well-Architected** | Framework de bonnes pratiques cloud (AWS, Azure, GCP) |
